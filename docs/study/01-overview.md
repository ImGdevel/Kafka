# 01. Kafka 한 장 요약

## Kafka는 뭐 하는 시스템인가?
Kafka는 “메시지를 보내고 받는 큐”로도 보이지만, 더 정확히는 **이벤트 스트리밍 플랫폼**이다.

- **Producer(프로듀서)**: 이벤트(메시지)를 Kafka로 보낸다.
- **Kafka Broker(브로커)**: 메시지를 저장/전달하는 서버 프로세스다.
- **Consumer(컨슈머)**: Kafka에서 메시지를 읽는다.

Kafka는 “서버가 메시지를 메모리에서 잠깐 들고 있다가 전달”하는 방식이 아니라, 기본적으로 **디스크에 로그처럼 저장**하고(=Commit Log) 컨슈머가 필요할 때 읽는다.

## 언제 Kafka를 쓰나?
- 서비스 간 “무슨 일이 일어났는지(이벤트)”를 전달하고 싶을 때
- 트래픽이 들쑥날쑥해도 안정적으로 처리하고 싶을 때(버퍼링)
- 같은 이벤트를 여러 시스템이 각자 소비해야 할 때(예: 알림/추천/적재)

## Kafka의 중요한 특징 3가지
1. **내구성**: 메시지를 디스크에 저장하고 일정 기간 보관한다(리텐션).
2. **확장성**: 토픽을 파티션으로 나눠 병렬 처리한다.
3. **재처리 가능**: 컨슈머가 “어디까지 읽었는지(오프셋)”만 기억하면 다시 읽을 수 있다.

## 아주 간단한 그림
```
Producer ---> [ Topic (Partition 0,1,2...) ] ---> Consumer Group
                 (Kafka broker에 저장)
```

## “공부할 때 꼭 잡아야 하는 질문”
- 토픽/파티션은 왜 필요한가? (성능/병렬성/순서 보장 범위)
- “정확히 한 번(Exactly-once)”은 언제 가능한가?
- 오프셋 커밋은 언제/어떻게 해야 안전한가?
